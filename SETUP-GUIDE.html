Fluence — Master Setup Guide

Go from "uploaded files" to "fully live, real app" in about 30 minutes. Written for a zero-code founder.

What you'll set up

1. Google Sheet (your database) 2. Apps Script backend (the API) 3. Google Form (brand onboarding) 4. Firebase Phone Auth (real OTP, free) 5. Paste keys into both HTML files 6. Re-deploy to Netlify Final checklist FAQ / troubleshooting
Before you begin. Keep this tab open. Open another tab for each service as you go. Everything is free to start.

1Google Sheet — your database

The whole backend lives in one Google Sheet with three tabs. This is where deals, brand applications, and creators are stored.

  1. Go to sheets.google.com → + Blank.
  2. Rename the file to Fluence DB.
  3. At the bottom, rename Sheet1 to Deals. Then click + twice to add two more tabs: Brands and Creators.
  4. Look at the URL. It will look like: 
    https://docs.google.com/spreadsheets/d/1AbCdEf...xyz/edit#gid=0
    Copy the bold part — that's your Sheet ID. Save it in a scratchpad; you'll paste it in Step 2.
Don't add headers manually. The Apps Script in Step 2 creates headers for you automatically.

2Apps Script — the backend API

Apps Script is free Google code that runs in the cloud. It turns your sheet into a REST API that the admin panel and creator app can talk to.

  1. In the Fluence DB sheet: Extensions → Apps Script. A new tab opens.
  2. Delete whatever code is there by default (the empty function myFunction).
  3. In your Fluence folder, open backend/google-apps-script.gs. Copy all of its contents (Cmd/Ctrl + A, then Cmd/Ctrl + C).
  4. Paste it into the Apps Script editor.
  5. At the top of the pasted code, find: 
    const SHEET_ID = "PASTE_YOUR_SHEET_ID_HERE";
const ADMIN_SECRET = "fluence_admin_2026_change_me";
    Replace the first value with the Sheet ID you copied in Step 1. Optionally change the admin secret (any random string).
  6. Click the floppy-disk icon to Save (name the project Fluence API).
  7. In the function dropdown at the top, select initializeSheets and click Run. Google will ask for permissions → Review permissions → pick your account → Advanced → Go to Fluence API (unsafe) → Allow. (It says "unsafe" only because you're the developer; it's your own code.)
  8. Go back to your sheet — the three tabs are now filled with headers, and Deals has one sample deal.
  9. In Apps Script, click Deploy → New deployment → ⚙ (gear) → Web app. Fill in:
    • Description: Fluence API v1
    • Execute as: Me
    • Who has access: Anyone
    Click Deploy. Approve access again if asked.
  10. You'll see a Web app URL ending in /exec. Copy it. This is your API_URL. Save it in the scratchpad.
Every time you change the Apps Script code, click Deploy → Manage deployments → ✏️ (pencil) → Version: New version → Deploy. The URL stays the same.

3Google Form — brand onboarding

Brands fill out this form, responses flow into your Brands tab as "pending", and you review/approve them in the admin panel.

  1. Go to forms.google.com → + Blank.
  2. Title: Fluence — Brand Onboarding (Gwalior).
  3. Add these questions in this exact order. The question title must match the "Field name" column exactly (lowercase with underscores) because Apps Script uses them as headers:
Field name (= Form question title)TypeExample label shown to brand
brand_nameShort answerWhat's your brand / business name?
contact_personShort answerYour name
emailShort answerEmail address
phoneShort answerWhatsApp number
categoryDropdownCategory (Food, Fashion, Salon, Gym, Events, Other)
budgetShort answerMonthly marketing budget (₹)
goalsParagraphWhat do you want from creator collabs?
websiteShort answerInstagram / website link
Exact naming matters. If the question title differs from the field name, the Apps Script onFormSubmit trigger won't know how to map it. If you prefer a prettier label for the brand, use the question description instead of changing the title.
  1. Click the Responses tab → the green Sheets icon → Select existing spreadsheet → Fluence DB. A new tab called Form Responses 1 will appear.
  2. Back in Apps Script, click Triggers (⏰ icon in left sidebar) → + Add Trigger:
    • Function: onFormSubmit
    • Event source: From spreadsheet
    • Event type: On form submit
    Click Save. Grant permissions.
  3. Click Send in the top-right of the form → copy the short link. That's the URL you'll share with brands via WhatsApp / Instagram bio / pitches.
  4. Test it yourself: submit a fake brand. Within 5 seconds it should show up in the Brands tab with status pending, and in the admin panel under Brand applications.

4Firebase Phone Auth — real OTP, free

Firebase's free tier gives you 10,000 SMS verifications per month in India. More than enough to start.

  1. Go to console.firebase.google.com → Add project → name it Fluence → disable Google Analytics (optional) → Create.
  2. In the left sidebar, click Build → Authentication → Get started.
  3. Click Sign-in method tab → Phone → toggle Enable → Save.
  4. Go to ⚙ Project settings → General. Scroll to Your apps. Click the </> Web icon. Register the app with nickname Fluence Web → skip hosting → Register.
  5. You'll see a firebaseConfig block. Copy the values — you need these five:
apiKey:     "AIza..."
authDomain: "fluence-xxxx.firebaseapp.com"
projectId:  "fluence-xxxx"
appId:      "1:1234:web:abc..."
messagingSenderId: "1234..."
  1. In Authentication → Settings tab → Authorized domains → Add domain → add your Netlify domain (e.g. fluence-app.netlify.app) and also localhost for testing.
No billing required. You stay on the free Spark plan. You only get charged if you ever exceed 10K SMS/month, which at Gwalior scale is years away.

5Paste the keys into both HTML files

Now wire everything together. Open your Fluence folder in Finder / File Explorer.

A) In index.html (the creator app)

Find the block near the top that says window.FLUENCE = { and fill it in:

window.FLUENCE = {
  API_URL: "https://script.google.com/macros/s/AKfyc.../exec",
  FIREBASE_CONFIG: {
    apiKey: "AIza...",
    authDomain: "fluence-xxxx.firebaseapp.com",
    projectId: "fluence-xxxx",
    appId: "1:1234:web:abc...",
    messagingSenderId: "1234..."
  },
  BRAND_ONBOARD_URL: "https://forms.gle/YOUR_FORM_ID"
};

B) In admin-panel.html

Find the block at the top that says window.FLUENCE_ADMIN = { and fill in:

window.FLUENCE_ADMIN = {
  API_URL: "https://script.google.com/macros/s/AKfyc.../exec",
  ADMIN_SECRET: "fluence_admin_2026_change_me"
};
ADMIN_SECRET must match whatever you set at the top of the Apps Script (Step 2). Otherwise the admin panel will get rejected by the backend.

Save both files. Open admin-panel.html by double-clicking it — you should see Backend live in the top-right corner.

6Re-deploy to Netlify

You already deployed once. Now that index.html has real keys, push the update:

  1. Copy the updated index.html into the deploy/ folder in Fluence (overwriting the old one).
  2. Go to app.netlify.com → your Fluence site → Deploys tab.
  3. Drag the deploy folder onto the "Drag and drop your site folder here" drop zone. Netlify publishes in ~10 seconds.
  4. Open your live URL on your phone. Sign up with your real number — you should receive a real SMS OTP.
You're live. Deals you post in admin-panel appear instantly in the creator app. Brand Form submissions show up in the approval queue. OTP is real.

?FAQ / troubleshooting

Admin panel says "Not connected"

You haven't set API_URL in admin-panel.html, or you set it to the wrong URL. Make sure it ends in /exec. Also confirm the Apps Script deployment has "Who has access: Anyone".

Admin panel says "Cannot reach backend"

URL is set but calls are failing. Usual causes:

  • You re-deployed Apps Script but clicked New deployment instead of Manage deployments → Edit → New version — generating a fresh URL. Copy the new URL back into admin-panel.html.
  • ADMIN_SECRET in HTML doesn't match the one in Apps Script.

Form submissions aren't showing up in Brand applications

  • Check Apps Script → Triggers — is onFormSubmit listed? Re-add it if not.
  • Open the Form Responses 1 tab in the sheet. If responses are there but not in Brands, the Apps Script code might have errored. In Apps Script → Executions you'll see red error rows — click one and read the message.
  • Common cause: the form question title doesn't match the field name (e.g. "Brand Name" instead of brand_name).

OTP isn't arriving

  • Is your Netlify domain in Firebase → Authorized domains?
  • Open the page on your phone; is the reCAPTCHA widget appearing? If no, check the browser console.
  • Phone number must start with +91 followed by 10 digits.
  • First SMS can take up to 60 seconds. Also check the spam SMS folder.

I want to change the admin secret later

Change it in both places: top of the Apps Script file (then redeploy the web app version) and top of admin-panel.html. Creators' sign-in doesn't use the secret, so they're unaffected.

How do I back up the data?

It's already in Google Sheets — automatic history. Also: File → Download → .xlsx to keep a local copy whenever you want.
