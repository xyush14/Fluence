🔥 Firebase + Google OAuth Setup

5-minute setup to get creators signing in with Google

Step 1️⃣: Create Firebase Project

A

Go to Firebase Console

Visit console.firebase.google.com

B

Create New Project

  • Click "Create Project"
  • Name: fluence
  • Disable Analytics (optional)
  • Click "Create"
C

Get Firebase Config

Once project is created:

  • Click gear icon ⚙️ (top-right) → "Project Settings"
  • Scroll down → "Your apps"
  • Click web icon </>
  • App name: fluence-web
  • Click "Register app"

Copy the firebaseConfig object (the blue box with apiKey, authDomain, etc).

⚠️ SAVE THIS! You'll need it in 5 minutes.
D

Enable Google Sign-In

  • In Firebase console, go to "Authentication" (left sidebar)
  • Click "Sign-in method"
  • Enable "Google" (toggle ON)
  • Save

Step 2️⃣: Get Google Client ID

A

Go to Google Cloud Console

Visit console.cloud.google.com

B

Create OAuth Credentials

  • Left sidebar → "APIs & Services" → "Credentials"
  • Click "Create Credentials" → "OAuth 2.0 Client IDs"
  • Application type: "Web application"
  • Name: Fluence Web
C

Add Authorized Origins

Under "Authorized JavaScript origins", add:

  • https://fluence.lol
  • https://app.fluence.lol
  • http://localhost:3000 (for testing)

Click "Create"

D

Copy Your Client ID

Copy the "Client ID" (looks like: 123456789-abc...def.apps.googleusercontent.com)

⚠️ SAVE THIS TOO!

Step 3️⃣: Paste into App

You now have 2 things to paste:

What Where to find Where to paste
firebaseConfig (JSON object) Firebase Console → Project Settings app-with-oauth.html line 29-36
Google Client ID (string) Google Cloud Console → Credentials app-with-oauth.html line 273
In app-with-oauth.html, find these 2 lines and replace:

Line 29-36 (Firebase Config):
Replace the entire firebaseConfig object from your Firebase project settings.

Line 273 (Google Client ID):
Replace YOUR_GOOGLE_CLIENT_ID.apps.googleusercontent.com with your actual Client ID

Step 4️⃣: Deploy to Netlify

1

Update your GitHub repo

# In your fluence folder:
cp app-with-oauth.html app/index.html
git add app/index.html
git commit -m "Add Google OAuth authentication"
git push origin main
2

Netlify auto-deploys

Your changes go live automatically in 2-5 minutes.

Check: app.fluence.lol should now show the updated app with "Sign in with Google" button

✅ Testing

Once deployed, test the flow:
  1. Visit app.fluence.lol
  2. Click "Sign in with Google"
  3. Log in with a Google account (your Gmail or any Google account)
  4. Fill in name, city, Instagram/YouTube (optional)
  5. Click "Continue to Deals"
  6. You should see mock deals (Cafe Mocha, FitHub Gym, etc.)
  7. Reload page — you stay logged in (data saved in browser)

❓ Troubleshooting

Problem Solution
"Sign in with Google" button doesn't appear Check that firebaseConfig is valid (check browser console for errors)
Sign-in clicks but nothing happens Check that Client ID is correctly pasted (no extra spaces)
"Origin not allowed" error Add your domain to Google Cloud Console → Credentials → Authorized origins
Profile doesn't save after refresh Normal for now — we're using browser localStorage. Real database coming next.
